# 概述

# 基本概念

计算机网络（简称网络）：由若干结点和连接这些结点的链路组成。

互连网（internet）：一个通用名词，不同网络之间通过路由器互连起来，构成了一个覆盖范围更大的计算机网络

理解

网络是把许多计算机连接在一起，而互联网是把许多网络连接在一起

互联网（Internet）：一个专用名词，指当前全球最大的、开放的、由众多网络相互连接而成的特定互连网，采用TCP/IP协议族作为通信的规则

ISP：互联网服务提供商，任何机构和个人只要向某个ISP交纳相应费用，就能获取所需的IP地址的使用权，所谓“上网”。

网络体系结构：计算机网络的各层及其协议的集合。

网络协议：为进行网络中的数据交换而建立的规则、标准、约定称为网络协议。由语法、语义、同步三个要素组成。

DETAILS

语法：数据与控制信息的结构或格式

语义：需要发出何种控制信息，完成何种动作，做出何种响应

同步：事件实现顺序的详细说明

协议：控制两个对等实体进行通信的规则，能够向上一层提供服务，实现本层服务还需下面一层提供服务。

理解

在协议的控制下，两个对等实体间的通信使得本层能够向上一层服务。

协议的实现保证了能够向上一层提供服务，使用本层服务的实体只能看见服务，无法看见下面的协议；

协议是水平的，服务是垂直的，上层使用下层所提供的服务必须通过和下层交换一些命令

TCP/IP协议族：特点为上下两头大而中间小，可以为各种应用提供服务，允许IP协议在各式各样的网络构成的互联网上运行

# 互联网的组成

互联网由边缘部分（所有连接在互联网上的主机）和核心部分（由大量网络和连接这些网络的路由器）组成

# 边缘部分

主机又称为端，在网络边缘的端与端通信由两种方式：C/S方式、P2P方式

# 核心部分

在核心部分起特殊作用的是路由器，其任务是转发收到的分组，这是核心部分最重要的功能

电路交换：整个报文的比特流连续从源点直达终点。通话的时间内，两个用户始终占用着端到端的资源

报文交换：一整个报文先到相邻结点，全部存储下来后再转发到下一个结点

分组交换：单个分组传送到相邻结点，存储下来转发到下一个结点。采用存储转发技术。

# 网络的分层

# 分层的好处

各层之间是独立的：某一层不需要知道下一层是如何实现的，而仅需要知道该层提供的服务。

灵活性好。当任何一层发生变化时，只要对某一层作修改，保持层间接口不变，对上下层均不产生影响

# OSI七层协议

# 物理层

功能：底层数据传输

# 数据链路层

功能：定义数据的基本格式，定义如何标识：如定义网卡MAC地址

# 网络层

功能：定义IP编址，定义路由功能

# 传输层

功能：端到端传输数据的基本功能

# 会话层

功能：控制应用程序之间的会话能力

# 表示层

功能：数据格式标识，基本压缩加密的功能

# 应用层

功能：各种应用软件

# 五层协议体系结构

# 应用层

任务：交互的数据单元称为报文。通过应用进程间的交互来完成特定网络应用，定义的是应用进程间通信和交互的规则。这里的进程指的是主机中正在运行的程序

# 运输层

任务：负责向两台主机中进程之间的通信提供通用的数据传输服务

# 网络层

任务：负责为分组交换网上的不同主机提供通信服务，把运输层的报文段封装成IP数据报，包装成分组，能够找到目的站，交付给目的站的运输层。

# 数据链路层

任务：属于计算机网络的低层。将网络层交下来的IP数据包组装成帧，在相邻两个结点之间的链路上传送帧

# 物理层

任务：用数据单位比特（0101的比特流）在发送方和接收方之间出传送，透明的传送比特流

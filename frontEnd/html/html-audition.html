# script标签中的async和defer

普通script，执行到标签处，dom渲染进程让出来给js加载然后执行，所以会让用户体验不好

defer script：执行到外部脚本处，会立刻加载，不会阻塞dom渲染，渲染完再执行加载完的脚本，就像标签放在body的下面；表示脚本可以延迟到文档完全被解析和显示之后再执行，会按照原本的顺序执行，只适用于外部脚本，会在DOMContentOnloaded事件之前加载

async script：执行到外部脚本处，会立刻下载，下载完就执行，所以很有可能不是按原本顺序执行；执行过程也会让dom进程让出来，但是不会用加载的时间；表示应该立即开始下载脚本，但不能阻止其他页面动作，不能保证有序，只适用于外部脚本

DETAILS

DOMContentLoaded事件：当纯HTML被完全加载和解析时触发，不必等待样式表、图片等资源的加载

onLoad事件：页面全部内容加载完毕后执行

script标签的其他属性

src：是一个url，指向的资源可以跟包含的HTML不在同一个域中

type：值为module，代码会被当成ES6模块

script标签放在body前导致页面渲染明显延迟，屏幕空白的时间延长

# src和href的区别

src：指向请求外部资源的来源地址，指向的内容会嵌入到文档中当前标签所在的位置。当浏览器解析到该元素时，会将资源下载并应用到文档里，暂停其他资源的下载，直到资源加载、编译、执行完毕

href：超链接，指向网络资源，当浏览器识别到它指向的文件时，就会并行下载资源，不会停止对当前文档的处理

# a标签除了导航功能以外还有什么作用？

最常用的是用来做锚点和文件下载

锚点可以点击之后快速定位到页面中的某个位置；下载的原理是a标签对应的资源浏览器无法解析，浏览器选择下载下来

# HTML attribute和DOM proprerty区别

获取attribute返回的值是string类型、而dom properties返回的值可以是任意类型

attribute初始化了property，在DOM节点创建的时候，HTML上的属性初始化了dom property

更改property值，attribute值不变

更改attribute值，相当于再初始化property值

attribute: DomObj.attributes获取HTML 属性，好实践是修改property，而不是setAttributes去修改或者getAttributes获取

# iframe元素

iframe元素：可以指定src（内框架的地址，可以是页面地址，也可以是图片的地址）

拿到iframe框架后，iframe.contentWindow拿到iframe里的window对象；

src同域的话，父页面可以对子页面进行改写；不同域，父页面没有权利改动子页面，但可以实现页面的跳转

iframe指定name属性，用window.frames[name]访问这个iframe

用possmessage进行跨域通信，iframe如何和外面框架通信？保持一致？答案是用postmessage。在组件中对iframe这个标签用ref获取，监听message事件，然后用ref获取iframe使用postmessage方法，possmessage("string", targetSrc)

# img标签的属性

src、alt、importance（下载资源时相对重要性/优先级）、loading（应当如何加载该图像）、width、height
